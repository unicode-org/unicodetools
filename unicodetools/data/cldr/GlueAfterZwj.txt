# Glue_After_Zwj is a binary property used to customize segmentation (UAX #29 and UAX #14)
# so that possible future emoji zwj sequences will not break grapheme clusters, words, or lines.
# For users, the bad effects from breaking too much around ZWJ are far worse than those from
# breaking too little around ZWJ. So while the rules here may be overly inclusive, that is not
# expected to cause any real problems for users.
#
# For Unicode 9.0, the minimal changes were made for the current ZWJ sequences.
# However, it is expected that many more ZWJ sequences may be encoded;
# it was too late for Unicode 9.0 to be adjusted to compensate.
#
# =====
# USAGE
#
# Implementations may use this file to adjust segmentation by customizing the following rules:
#
# OLD (Unicode 9.0)
#
#	GB11	ZWJ	Ã—	(Glue_After_Zwj | EBG)
#	WB3c	ZWJ	Ã—	(Glue_After_Zwj | EBG)
#	LB8a	ZWJ	Ã—	(ID | EB | EM)
#
# CUSTOMIZATION
#
#	let Glue_After_Zwj be defined as in this file
#	let Emoji = \p{Emoji=Yes}
#
#	GB11	ZWJ	Ã—	(Glue_After_Zwj | Emoji)
#	WB3c	ZWJ	Ã—	(Glue_After_Zwj | Emoji)
#	LB8a	ZWJ	Ã—	(ID | Glue_After_Zwj | Emoji)
#
# =====
# Review Notes 
# 1. More conservative implementations could use the following, more restrictive rules:
#
#	let Precontext = (Glue_After_Zwj | Emoji) emoji_variation_selector? non_spacing_mark*
#	let PrecontextID = (ID | Glue_After_Zwj | Emoji) emoji_variation_selector? non_spacing_mark*
#
#	GB11'	Precontext ZWJ	Ã—	(Glue_After_Zwj | Emoji)
#	WB3c'	Precontext ZWJ	Ã—	(Glue_After_Zwj | Emoji)
#	LB8a'	PrecontextID ZWJ	Ã— (ID | Glue_After_Zwj | Emoji)
#
# 2. The ID is retained in LB8a'. The main reason for its inclusion was to encompass as many Emoji
#    characters as possible. So it could be removed from LB8a'. It is only retained here to
#    allow for as much compatibility with Unicode 9.0 as possible.
#
# =====

# Field 1 indicates that the character(s) listed in Field 0 have the value Glue_After_Zwj=Yes
# All other characters have the value Glue_After_Zwj=No
